# WorldQuant 字段速查

## 1）先看目标：你想优化哪个主字段？

### 想提升 `Fitness`
优先看：`Sharpe`、`Turnover`、`Drawdown`、`Neutralization`、`Pasteurize`、`Truncation`、`Nan Handling`、`Unit Handling`

### 想提升 `Sharpe`
优先看：`Drawdown`、`Neutralization`、`Pasteurize`、`Decay`、`Truncation`、`ZScore(a)`、`Rank(a)`

### 想提升 `Sub-universe Sharpe`

优先看：`Decay`、`Window`、`Neutralization`、`Universe`、`Truncation`、`Rank(a)`

与 `Sharpe` 相比：去掉了 `Drawdown`、`Pasteurize`、`ZScore(a)`，新增了 `Window`、`Universe`

### 想降低 `Turnover`

优先看：`Decay`、`Delay`、`Weight`、`Truncation`、`Universe`

### 想提升 `PnL` / `Returns`
优先看：`Long`、`Short`、`Region`、`Universe`、`Delay`、`CLV`、`ts_mean`、`ts_sum`

---

## 2）Setting

### **Language**

定义：选择因子表达式的编写语言。

常见下拉选项：

- **Fast Expression**
  - 高性能表达式语言
  - 运行速度快
  - 支持大部分算子
  - 常用默认选项
- **Expression**
  - 标准表达式语言
  - 运算速度较慢
  - 旧版本兼容用途

------

### **Instrument Type**

定义：选择交易资产类别。

常见下拉选项：

- **Equity**
  - 股票
  - 最常用资产类型
- **ETF**
  - 交易型基金
- **Futures**
  - 期货
- **Forex**
  - 外汇
- **Crypto**
  - 加密货币（部分平台支持）

不同资产类型会影响数据结构、可用字段和交易规则。

------

### **Region**

定义：数据所属市场区域。

常见下拉选项：

- **USA**
- **China**
- **Europe**
- **Japan**
- **Global**
- **Asia ex-Japan**

不同地区影响股票池范围、行业分类体系和市场特征。

------

### **Universe**

定义：选择参与排序和回测的股票集合。

常见下拉选项：

- **TOP500**
  - 大盘股为主
  - 流动性高
  - 稳定性强
- **TOP1000**
- **TOP2000**
- **TOP3000**
  - 包含更多中小盘股
  - Alpha 机会更多
  - 波动更大
- **ALL**
  - 所有股票
  - 噪音较大

Universe 越大，波动和潜在 Alpha 空间通常越高。

------

### **Delay**

定义：信号生成与交易执行之间的时间差。

常见选项：

- **0**
  - 当天信号当天交易
  - 存在未来函数风险
- **1**
  - 今天生成信号
  - 明天执行交易
  - 常用标准设置
- **2**
  - 延迟两天执行
  - 更保守稳健

------

### **Neutralization**

定义：消除系统性风险暴露。

常见选项：

- **None**
  - 不进行中性化
- **Market**
  - 去除市场整体 Beta
- **Sector**
  - 一级行业中性
- **Industry**
  - 二级行业中性
- **Subindustry**
  - 子行业中性
  - 最严格的行业中性方式

中性化越细，风险暴露控制越严格。

------

### **Decay**

定义：信号的时间加权平滑程度。

常见数值：

- **0**
  - 不平滑
  - 最敏感
- **2–4**
  - 短期平滑
- **5–10**
  - 中期平滑
- **20+**
  - 长期信号

数值越大，信号越平滑，换手率通常越低。

------

### **Truncation**

定义：限制单只股票的最大权重比例。

常见数值：

- **0**
  - 不限制
- **0.01–0.05**
  - 严格限制
- **0.08**
  - 常见标准值
- **0.1**
  - 较宽松限制

用于控制极端值对组合的影响。

------

### **Pasteurization**

定义：是否对异常数据进行清洗。

常见选项：

- **On**
  - 去除异常值
  - 提高稳定性
- **Off**
  - 不做清洗
  - 可能增加噪音

------

**Unit Handling**

定义：是否检查数据单位一致性。

常见选项：

- **Verify**
  - 验证单位
  - 防止单位错误
- **Ignore**
  - 忽略单位检查
  - 可能产生逻辑问题

------

### **NaN Handling**

定义：缺失值处理方式。

常见选项：

- **Off**
  - 不处理缺失值
- **Zero**
  - 用 0 填充
- **Backfill**
  - 向前填充
- **Forward Fill**
  - 使用最近历史值填充

不同处理方式会影响因子稳定性。

------



### `Test Period`

**定义**：在总回测区间内，切出一段时间作为样本外验证期（OS），隐藏其 PnL 结果。

**输入方式**：手动填写 `YEARS` + `MONTHS`

**Period 切换**：

- **IS**（In Sample）：样本内训练期，正常可见，用于调参
- **OS**（Out of Sample）：样本外验证期，默认隐藏，点击 OS 按钮查看

**回测结构**：
- 总回测时长 = IS 期 + OS 期
- Test Period 设置的时长 = OS 期长度（从末尾切出）

**常见值**：

| 设置               | 效果                      |
| ------------------ | ------------------------- |
| `0 Years 0 Months` | 不切验证集，全部为 IS     |
| `1 Year`           | 末尾 1 年为 OS，其余为 IS |
| `2 Years`          | 末尾 2 年为 OS，其余为 IS |

**设计目的**：防止过拟合，OS 相当于封存的验证答案，
最终点开 OS 查看，检验因子真实泛化能力。

> ⚠️ 注意：`Test Period` 不是回测总时长，
> 而是从总回测中**切出隐藏**的验证段长度。



## 3）字段卡片（每个字段 4 行）

### `Sharpe`
- **是什么**：单位风险收益比
- **干什么**：衡量“赚得稳不稳”
- **主要影响**：`Fitness`
- **例子**：`Sharpe = 1.8`（通常优于 `Sharpe = 0.7`）

### `Turnover`
- **是什么**：换手率（调仓频率）
- **干什么**：衡量交易活跃度/成本压力
- **主要影响**：`Fitness`
- **例子**：`Turnover = 0.35` 表示平均每天换仓约 35%

### `Fitness`
- **是什么**：综合评分（收益+稳定+成本+约束）
- **干什么**：决定是否达标
- **主要影响**：被其他字段共同决定
- **例子**：`Fitness = 1.12` 且 `cutoff = 1.0` → 通过

### `PnL`
- **是什么**：累计盈亏曲线
- **干什么**：看赚钱连续性与回撤
- **主要影响**：`Sharpe`、`Fitness`
- **例子**：1 年累计从 0 增长到 `+12%`

### `Returns`
- **是什么**：收益率（累计/年化）
- **干什么**：看“赚得多不多”
- **主要影响**：`PnL`、`Sharpe`、`Fitness`
- **例子**：`Annualized Returns = 14.5%`

### `Drawdown`
- **是什么**：最大回撤
- **干什么**：看抗风险能力
- **主要影响**：`Sharpe`、`Fitness`
- **例子**：`Max Drawdown = -6.2%`

### `Margin`
- **是什么**：单位交易收益效率
- **干什么**：看交易值不值
- **主要影响**：`PnL`、`Returns`、`Fitness`
- **例子**：两策略收益相同，`Margin` 高的通常更优

---

### `Region`
- **是什么**：市场（USA/CHN…）
- **干什么**：决定市场结构和风格
- **主要影响**：`PnL`、`Returns`、`Sharpe`、`Fitness`
- **例子**：`Region = USA`

### `Universe`
- **是什么**：股票池范围（TOP3000…）
- **干什么**：决定可交易样本与流动性
- **主要影响**：全部主字段（尤其 `Turnover`、`Returns`）
- **例子**：`Universe = TOP3000`

### `Delay`
- **是什么**：信号延迟执行天数
- **干什么**：防未来函数，贴近实盘
- **主要影响**：`PnL`、`Returns`、`Turnover`、`Sharpe`、`Fitness`
- **例子**：`Delay = 1`（用 T 日信号在 T+1 交易）

### `Decay`
- **是什么**：仓位惯性/平滑
- **干什么**：降低调仓速度
- **主要影响**：`Turnover`（最直接），其次 `Sharpe`、`Fitness`
- **例子**：`Decay = 10` 通常比 `Decay = 1` 换手更低

### `Truncation`
- **是什么**：单票权重上限
- **干什么**：防持仓过度集中
- **主要影响**：`Sharpe`、`Fitness`、`PnL`
- **例子**：`Truncation = 0.08`（单票绝对权重不超过 8%）

### `Neutralization`
- **是什么**：行业/市值等风险中性化
- **干什么**：去掉非目标风险暴露
- **主要影响**：`Sharpe`、`Fitness`（常见提升）
- **例子**：`Neutralization = Sector`（按行业中性化）

### `Pasteurize`
- **是什么**：异常值抑制
- **干什么**：降低极端值干扰
- **主要影响**：`Sharpe`、`Fitness`
- **例子**：开启后，极端分数从 20 缩到可控范围

### `Nan Handling`
- **是什么**：缺失值处理
- **干什么**：防信号断裂
- **主要影响**：`Fitness`、`Sharpe`、`PnL`
- **例子**：将 NaN 设为 0 或用截面均值填充

### `Unit Handling`
- **是什么**：量纲/单位对齐
- **干什么**：避免混算失真
- **主要影响**：`Sharpe`、`Fitness`、`Returns`
- **例子**：将“百分比”和“绝对值”先标准化再合成

---

### `Weight`
- **是什么**：单票持仓权重（正多负空）
- **干什么**：控制仓位分配
- **主要影响**：`Turnover`、`PnL`、`Returns`、`Fitness`
- **例子**：`AAPL = +0.02`，`TSLA = -0.015`

### `Long`
- **是什么**：多头暴露
- **干什么**：从上涨中获利
- **主要影响**：`PnL`、`Returns`
- **例子**：买入评分最高的前 20% 股票

### `Short`
- **是什么**：空头暴露
- **干什么**：从下跌中获利/对冲
- **主要影响**：`PnL`、`Returns`、`Sharpe`
- **例子**：卖空评分最低的后 20% 股票

---

### `CLV`
- **是什么**：收盘在日内高低区间的位置强弱
- **干什么**：判断尾盘强弱信号
- **主要影响**：`Returns`、`PnL`、`Sharpe`
- **例子**：\((2*close - high - low)/(high-low)\)，若 `0.8` 表示收盘接近最高

### `Rank(a)`
- **是什么**：横截面排序分位化
- **干什么**：做相对强弱比较
- **主要影响**：`Sharpe`、`Fitness`、`Returns`
- **例子**：当日 1000 只股票中第 900 名，`rank(a)≈0.9`

### `ZScore(a)`
- **是什么**：标准化（偏离均值几个标准差）
- **干什么**：统一尺度、识别异常
- **主要影响**：`Sharpe`、`Fitness`
- **例子**：`zscore(a)=1.5` 表示高于均值 1.5 个标准差

### `-ZScore(a)`
- **是什么**：`ZScore` 反向
- **干什么**：常用于反转逻辑
- **主要影响**：`Returns`、`PnL`、`Sharpe`
- **例子**：若 `zscore(a)=1.2`，则 `-zscore(a)=-1.2`

### `ts_sum(x,d)`
- **是什么**：过去 d 天求和
- **干什么**：累计效应统计
- **主要影响**：`Returns`、`PnL`
- **例子**：`ts_sum(returns>0?1:0,250)` = 250 天里上涨天数

### `ts_mean(x,d)`
- **是什么**：过去 d 天均值
- **干什么**：平滑噪声
- **主要影响**：`Sharpe`、`Turnover`、`Returns`
- **例子**：`ts_mean(close,10)` = 10 日均收盘价





# 奖励规则——顾问审核

原句：  
**正式顾问一天的收入从 1-120 美元不等，主要取决于你当天交的因子的总体质量，value factor, osmosis rank 和 prod correlation 等因素。**

---

### 字段定义

### **Base**

定义：成为顾问后，每日提交 Alpha 所获得的基础报酬。

影响因素：

- 质量（Quality）
  - 最重要的因素
  - 直接影响排名与 base 高低
- Value Factor（VF）
  - 价值因子
  - 初始值为 0.5，随表现动态更新
- Osmosis Rank
  - 渗透排名
- Prod Correlation
  - 与已有因子的相关性

Base 并非单纯由单个 Alpha 质量决定，而是综合多个维度计算得出。

------

### **单交 vs 多交策略**

定义：同一天可选择提交单个或多个 Alpha，提交方式影响 base 计算逻辑。

- 单独提交（1个）
  - 以该 Alpha 单独质量参与排名
  - 适合对该因子质量把握较高时使用
- 组合提交（多个）
  - 系统将多个 Alpha 组成 Portfolio
  - 以**综合质量**统一排名
  - 若某 Alpha 质量偏低，会**拖累整体 base**

注意事项：

- 质量存疑的 Alpha 建议暂不提交，避免拉低整体表现
- 不可依赖数量堆积来提升 base
  - 低质量 Alpha 数量再多，也只会拉低综合质量
  - 质量才是决定 base 高低的核心

------

### **Base 与收益对应参考**

定义：base 数值与实际每日收益之间的对应关系。

关键参考数据：

- base = 1.7，每天提交 1 个 Alpha
  - 实际收益约 **1.5 美金/天**
  - 可理解为**及格线水平**
- 日收益低于 1.5 美金
  - 表现低于及格线，需优化 Alpha 质量
- 日收益高于 1.5 美金
  - 表现高于及格线
  - 该 Alpha **值得深入学习**
  - 可考虑**泛化为模板**，批量复用

收益受提交数量与质量共同影响，单日单交时参考意义最强。

------

### **Value Factor（VF）参考标准**

定义：衡量 Alpha 实际产出价值的因子，影响长期收益。

关键节点：

- VF = 0.5

  - 初始起点

- VF 随 Alpha 表现

  动态更新

  - 表现越好，VF 提升越快
  - VF 越高，说明该 Alpha 实际价值越强

---

### 简要结论

日收入并非固定值，通常由以下因素共同决定：

**Income ~ f(alpha_quality, value_factor, osmosis_rank, prod_correlation, ...)**

即：**因子质量 + 账户系数 + 排名表现 + 与生产信号的独特性** 共同影响最终收入。
