💡 核心目标：找到"谁比谁强"的规律，多强的空弱的
❌ 直接用收益率
股票A +3%
股票B +2%
股票C +1%
股票D +50% ← 并购！
→ 组合被股票D主导！
✅ 用rank()
股票A rank=3 → 0.75
股票B rank=2 → 0.50
股票C rank=1 → 0.25
股票D rank=4 → 1.00
→ 权重均衡，极端值无法主导
核心感悟： rank() = 把原始数值转成排名（0到1之间），不管原始差距多大，第1名就是第1名。
// 为什么用负号？
rank(-returns)
→ 负号让排名翻转
→ 涨最多 = rank最低(≈0) → 做空（均值回归）
→ 跌最多 = rank最高(≈1) → 做多（均值回归）

// 操作：每个值 ÷ 所有值的绝对值之和

绝对值之和 = 0.50+0.36+0.21+0.07+0.07+0.21+0.36+0.50
= 2.28

各权重 ÷ 2.28 → 使得：
正数之和 = +0.50 → 做多 $1000万
负数之和 = -0.50 → 做空 $1000万
为什么要做： 不同Alpha公式量级不同（有的0.01，有的100），归一化后统一变成"比例"，就像考试要统一用百分制才能公平比较。
× $2000万 → 永远是 多$1000万 + 空$1000万
📐 通用公式（线性衰减加权平均）
最终信号 = ( 今天×d + 昨天×(d-1) + … + d天前×1 )
÷ ( 1+2+…+d = d×(d+1)/2 )
权重线性递减 · 今天始终是最高权重 d · d天前权重为 1
🔥 Decay = 3（快）— 总权重 = 3+2+1 = 6
今天 (t)    × 3
3/6 = 50.0% 昨天 (t-1) × 2
2/6 = 33.3% 前天 (t-2) × 1
1/6 = 16.7%
⚡ 今天权重高达 50%，信号对最新数据极度敏感，换手率高，每日调仓频繁，交易成本大
🐢 Decay = 10（慢）— 总权重 = 10+9+…+1 = 55
今天   (t)    × 10
10/55 = 18.2% 1天前 (t-1)  ×  9
9/55  = 16.4% 2天前 (t-2)  ×  8
8/55  = 14.5% 3天前 (t-3)  ×  7
7/55  = 12.7% 4天前 (t-4)  ×  6
6/55  = 10.9% 5天前 (t-5)  ×  5
5/55  =  9.1% 6天前 (t-6)  ×  4
4/55  =  7.3% 7天前 (t-7)  ×  3
3/55  =  5.5% 8天前 (t-8)  ×  2
2/55  =  3.6% 9天前 (t-9)  ×  1
1/55  =  1.8%
🛡️ 今天仅占 18.2%，过去10天共同决定信号，平滑稳定，换手率低，对单天噪音不敏感
🧮 平滑效果数值示例（Decay = 3）
原始3天信号：  t-2 = +0.40   t-1 = -0.20   t  = +0.60
// ❌ 不用 Decay（直接取今天值）
信号 = +0.60  ← 昨天是-0.20也不管

// ✅ 使用 Decay = 3
信号 = ( 3 × +0.60  +  2 × -0.20  +  1 × +0.40 ) ÷ 6
      = ( 1.80 − 0.40 + 0.40 ) ÷ 6
      = 1.80 ÷ 6
      = +0.30  ← 被昨天的负值拉低了一半
如果 +0.60 只是随机噪音，Decay 帮你避免了因噪音频繁调仓 ✅
⚡ 小 Decay（3～5）
✅ 信号更新快
✅ 能捕捉短期反转
❌ 换手率高，成本大
❌ 对噪音更敏感
🐢 大 Decay（10～20）
✅ 信号平滑稳定
✅ 换手率低，成本小
❌ 对新信息反应迟钝
❌ 可能错过短期机会
注意： Decay大 ≠ 当天数据不影响结果。当天仍然权重最高，只是被"稀释"了，历史数据也分到了权重。

为什么容易过拟合？
→ 参数改太多：用哪个指标？用几天？Decay多少？
→ 每改一次参数 = 多背了一遍答案
→ 改得越多，越"适配"历史，越不适应未来
IS/OS 的设计就是为了抓住"背答案"的行为
💡 核心概念一句话总结
📊rank()把数值转成排名，极端值不再主导结果
⚖️中性化减去均值，只赌相对强弱，不赌市场方向
📐归一化÷绝对值之和，统一量级方便比较
⏳Decay值小反应快，今天信号占比高；值大历史数据也有话语权
🔬IS/OSIS是历年真题，OS是真正考试，都好才是真Alpha
