# ts_rank(operating_income, 252)

# 改进1：加入股价，变成营业收益率
# ts_rank(operating_income / close, 252)

# Unit Handling : Verify
# Max Trade     : OFF
# Max Position  : OFF

# 演变过程：
# Step 1: 原版 liabilities/assets + Market中性化 → Sharpe 0.77 ❌
#         不同行业负债率差异巨大，Market中性化无法消除行业噪音
#
# Step 2: 改为 Subindustry中性化 → Sharpe 1.54 ✅
#         同子行业内比较，信号更纯净
#
# Step 3: 尝试 liabilities/sqrt(cap * assets) → Sharpe 1.33 ✅
#         分母引入市值，账面+市场双视角衡量杠杆
#         有单位WARNING但不影响提交

# 最终答案：混合杠杆率
# 负债 ÷ (市值×总资产的几何平均)
# 同子行业内，相对负债越重的公司，短期越跑赢
【已提交：2026-04-14 17:56】liabilities/sqrt(cap * assets)

### 5.短期情绪量稳定性 Short-Term Sentiment Volume Stability

```vala
# ========== Simulation Settings ==========
# Region        : USA
# Universe      : TOP3000
# Language      : Fast Expression
# Decay         : 0
# Delay         : 1
# Truncation    : 0.08
# Neutralization: Industry
# Pasteurization: On
# NaN Handling  : Off
# Unit Handling : Verify
# Max Trade     : OFF
# Max Position  : OFF
# =========================================

# 演变过程：
# Step 1: 原版 → Sharpe 1.82 ✅ 但 Fitness 0.45 ❌
#         -ts_std_dev(scl12_buzz, 10)
#         信号本身很强，但大市值股票权重不足导致 Fitness 低
#
# Step 2: 换 fast 数据源短窗口 → Sharpe 1.25 ✅ Sub-u Sharpe 0.40 ❌
#         -ts_std_dev(scl12_buzz_fast_d1, 7)
#         fast 版本对大市值股票信号反而更弱
#
# Step 3: fast + 原版结合 → Sharpe 1.23 ❌ Sub-u Sharpe 0.45 ❌
#         -ts_std_dev(scl12_buzz_fast_d1, 10) - ts_std_dev(scl12_buzz, 10)
#         全面下降，方向错误
#
# Step 4: 行业内标准化 → Sharpe 1.26 ✅ Sub-u Sharpe -0.08 ❌
#         -group_zscore(ts_std_dev(scl12_buzz, 10), industry)
#         行业标准化抹平了原有信号强度，适得其反
#
# Step 5: 市值加权 → Sharpe 1.55 ✅ Fitness 1.31 ✅ Sub-u Sharpe 0.88 ✅
#         -ts_std_dev(scl12_buzz, 10) * log(cap)
#         直接对大市值股票加权，Fitness 和 Sub-u 同时修复
#         7 PASS 0 FAIL ✅

# 最终答案：社交媒体热度波动率 × 市值加权
# buzz 过去10天标准差 → 反向 → 乘以 log(市值)
# 同行业内，热度越稳定的大市值股票，短期更跑赢

可提交，还未提交：
-ts_std_dev(scl12_buzz, 10) * log(cap)
```
